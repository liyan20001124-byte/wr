## 四、模块详细说明

### [1] 授权码捕获 (行 130-140)

当用户在 `login.live.com/oauth20_desktop.srf` 完成 OAuth 授权后，自动从 URL 参数中提取 `code`，保存到 `Config.code`，并关闭页面。

### [2] RewardsAuto 配置对象 (行 146-305)

| 属性 | 说明 |
|------|------|
| `ua.pc` / `ua.mobile` / `ua.app` | 三种设备的 User-Agent |
| `appConfig` | App 端配置：rewardsAppId、channel、offerIds |
| `searchPool` | 本地搜索词池（中英文混合，~50 个） |

### [3] Webhooks 通知渠道 (行 307-393)

5 个通知渠道，每个包含 `name`、`url`、`key`（从 GM_getValue 读取）、`msg`（消息模板）。

| 渠道 | API 端点 | 消息格式 |
|------|----------|----------|
| 企业微信 | `qyapi.weixin.qq.com/cgi-bin/webhook/send?key=` | text |
| 钉钉 | `oapi.dingtalk.com/robot/send?access_token=` | markdown |
| 飞书 | `open.feishu.cn/open-apis/bot/v2/hook/` | interactive card |
| PushMe | `push.i-i.me/?push_key=` | markdown |
| Bark | `api.day.app/` | markdown |

### [4] 悬浮窗 UI (行 395-554)

**当前问题**: 因 `@crontab` 存在，ScriptCat 只在后台沙箱运行，悬浮窗永远不会显示。

**设计**:
- `HAS_DOM` 守卫：`typeof document !== "undefined" && typeof document.createElement === "function"`
- CSS 通过 `document.createElement("style")` 注入（不使用 `GM_addStyle`，因 ScriptCat 沙箱不支持）
- 白色极简主题，参考 Rewards 助手
- 默认折叠（44px 圆形图标），点击展开
- 面板元素：等级、积分、PC搜索进度条、移动搜索进度条、阅读进度条、活动计数、授权码区域、操作按钮、日志区

**函数**:

| `mrPanelUpdateStats(info)` | 更新进度显示（noop 被覆盖） |

### [10] 入口引导 (行 2650-2753)

**前台模式** (`HAS_DOM = true`):
- 创建悬浮窗、绑定按钮事件
- 初始化数据、更新 UI
- 按钮触发任务（一键/搜索/阅读/活动/签到）

**后台模式** (`HAS_DOM = false`):
- cron 触发 → `init()` → 随机延迟 5-95 秒 → `runAll()`
- 跨标签搭便车检查
- 今日任务完成检查

## 七、任务执行流程

```
cron 触发 (每20分钟)
  │
  ├─ 后台模式 (HAS_DOM=false)
  │   └─ init()
  │       ├─ 检查今日任务是否全部完成
  │       ├─ 跨标签搭便车检查
  │       └─ 随机延迟 5-95秒 → runAll()
  │
  └─ 前台模式 (HAS_DOM=true)
      └─ 显示悬浮窗，等用户点击按钮

runAll():
  1. 地区检测 (checkRegion)
  2. Token 续期 (renewToken)
  3. 签入 (doSign) — App端 + PC端
  4. 阅读 (doRead) — 逐步完成，实时更新UI
  5. 活动 (doPromos) — 扫描卡片 → 领取
  6. 搜索 (doSearch) — PC + 移动，随机4-7次
  7. 每日活动 (doDailySet) — DOM 点击
  8. 二次扫描 (discoverCards) — 检查新解锁卡片
  9. 汇总通知 (getRewardsInfo + Webhook)
```

## 八、已知问题与限制

1. **@crontab 与悬浮窗冲突**: ScriptCat 中 `@crontab` 脚本只在后台沙箱运行，无 DOM，悬浮窗无法显示。需拆分前台/后台为两个脚本，或去掉 `@crontab` 改为纯前台模式。

2. **GM_addStyle 不可用**: ScriptCat 沙箱不支持 `GM_addStyle`，已改用 `document.createElement("style")`。

3. **getSearchQuotaFromAPI 已弃用**: v2.9.1 改用 `fetchDashboard()`，但 `getSearchQuotaFromAPI` 代码仍保留（未删除），实际不再被调用。

4. **移动端域名**: 当前 PC 和移动搜索都用 `www.bing.com`，Rewards 助手用 `cn.bing.com` 做移动搜索。

5. **阅读任务从 Dashboard 获取**: Dashboard API 的 `dailySetPromotions` 中不一定包含阅读任务信息（offerId 匹配），可能需要回退到 DAPI 查询。

6. **next-action 哈希**: 动态从页面提取，但如果页面结构变化或后台模式无法访问页面，则使用硬编码 fallback。
